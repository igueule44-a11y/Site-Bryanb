Ollama Cloud × Claude Code : le guide complet pour ne plus jamais être bloqué par tes tokens

Comment connecter Ollama Cloud à Claude Code et Hermès Agent pour ne plus jamais être bloqué par tes tokens, pas à pas et sans coder.

Si tu utilises Claude Code ou Hermès Agent, tu connais la douleur : tu tombes à court de tokens au pire moment. Ollama Cloud règle ça. Voici comment tout brancher en 10 minutes.

Pourquoi Ollama Cloud plutôt qu'un autre provider

Ollama Cloud te donne accès à des modèles open-source (Llama, Qwen, DeepSeek, Mistral) via une API simple, compatible OpenAI. Avantages :

  • Coût marginal quasi-nul sur les modèles locaux bien plus petits que Claude
  • Pas de rate limit agressif comme les API classiques
  • Confidentialité : tes données ne servent pas à entraîner le modèle
  • Variété : tu peux router entre un gros modèle (DeepSeek V3) pour les tâches complexes et un petit (Llama 3.1 8B) pour les tâches simples

Étape 1 : récupérer ta clé Ollama Cloud

Rends-toi sur ollama.com, crée un compte, va dans Settings → API Keys. Copie la clé au format ollama-....

Étape 2 : configurer Claude Code

Dans ton shell :

export ANTHROPIC_BASE_URL=https://ollama.com/v1
export ANTHROPIC_API_KEY=ollama-ta-cle-ici

Tu peux aussi créer un alias pour switcher rapidement :

# ~/.zshrc ou ~/.bashrc
alias cc-ollama='ANTHROPIC_BASE_URL=https://ollama.com/v1 ANTHROPIC_API_KEY=ollama-... claude'
alias cc-anthropic='claude'

Étape 3 : brancher Hermès Agent

Dans ~/.hermes/config.yaml, ajoute un custom provider :

providers:
  - name: ollama-cloud
    base_url: https://ollama.com/v1
    api_key: ollama-ta-cle
    models:
      - name: deepseek-v3
        context: 128000
      - name: llama-3.3-70b
        context: 128000

Tu peux ensuite router une partie de tes tâches vers ce provider dans les cron jobs ou directement à la main.

Étape 4 : la stratégie de bascule

La vraie force c'est le fallback chain : Claude Sonnet pour le code complexe, Ollama Cloud pour la recherche, le résumé, la classification. Ton coût par tâche baisse de 60-80% sans perte de qualité sur les tâches critiques.

Concrètement, dans un cron job Hermès :

model:
  primary: anthropic/claude-sonnet-4
  fallbacks:
    - ollama-cloud/deepseek-v3
    - ollama-cloud/llama-3.3-70b

Cas d'usage qui marchent bien sur Ollama Cloud

  • Recherche et résumé web (Llama 70B suffit)
  • Génération de première version d'article (DeepSeek V3)
  • Classification et tagging (petits modèles)
  • Analyse de logs et debugging simple

À éviter : génération de code React complexe ou raisonnement multi-étapes long — reste sur Claude Sonnet pour ça.
