📖 Voici toute l'histoire

Tu veux connaître le vrai contexte ? J'ai passé 3 heures à me battre contre des erreurs incompréhensibles pour configurer Ollama Cloud. Clique sur le bouton ci-dessous pour lire l'histoire complète, ou saute directement au guide pratique si tu veux juste que ça fonctionne.

Je documente publiquement mon apprentissage de l'IA depuis environ un mois, sans aucun background de développeur. Ce matin, je voulais juste faire une chose simple : configurer Ollama Cloud avec Hermès Agent et Claude Code.

Le piège du faux terminal

Tout a commencé avec une erreur : "aucune information de connexion valide trouvée." Mon réflexe a été d'aller corriger ça dans VS Code. Sauf qu'en réalité, je n'étais pas dans un vrai terminal — j'étais dans une conversation avec un modèle d'IA qui ressemble à un terminal. J'ai donc "configuré" un fichier qui n'a jamais réellement été modifié.

Le mot de passe qui a fui

En cherchant à débloquer la situation, j'ai copié-collé une clé d'accès directement dans une conversation. Résultat : elle a été automatiquement désactivée. J'ai continué à essayer de la faire fonctionner alors qu'elle était déjà morte.

L'erreur qui s'est incrustée

À un moment, j'ai enregistré une configuration qui pointait vers OpenRouter dans les réglages profonds de Hermès. Cette information s'est incrustée tellement profondément que même en supprimant les dossiers et en réinstallant depuis zéro, Hermès continuait à revenir vers OpenRouter.

Ce qui a fini par débloquer

Au final, ce ne sont pas les clics qui m'ont sauvé, mais deux commandes :

rm -rf ~/.hermes
hermes model

Leçons

• Vérifie où tu es vraiment avant de taper une commande
• Ne partage jamais une clé d'accès en clair
• Si une interface graphique bloque, essaie une commande simple
• Une configuration mal enregistrée peut survivre à une suppression

🚀 Guide Pratique

1️⃣ Créer un compte Ollama Cloud

  1. Va sur ollama.ai
  2. Crée un compte avec ton email
  3. Valide ton email
  4. Crée une nouvelle clé API et copie-la dans un endroit sûr

2️⃣ Configurer dans VS Code

  1. Installe l'extension Cline
  2. Va dans les paramètres
  3. Configure le fournisseur comme "Ollama Cloud"
  4. Colle ta clé API
  5. Choisis ton modèle

3️⃣ Configurer Hermès Agent (optionnel)

hermes config

Puis réponds à ces questions : Provider → ollama_cloud | API Key → (ta clé) | Model → (ton modèle)

⚠️ Pièges courants

Clé exposée ? Regénère-la immédiatement dans le dashboard Ollama.
Configuration buggée ? Utilise: rm -rf ~/.hermes puis hermes model
VS Code ne reconnaît pas Ollama ? Redémarre VS Code
Besoin de tester ta clé ? Elle fonctionne si tu reçois une réponse sans erreur
