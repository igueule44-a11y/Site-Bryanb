Je documente publiquement mon apprentissage de l'IA depuis environ un mois, sans aucun background de développeur. Ce matin, je voulais juste faire une chose simple : configurer un agent IA (Hermès Agent) pour qu'il utilise Ollama Cloud, un service que je paye déjà 20$/mois, à la fois sur mon assistant de code et sur mon agent perso.

Trois heures plus tard, j'avais l'impression de m'être battu contre un mur. Voici ce qui s'est passé, et ce que ça m'a appris.

Le piège du faux terminal

Tout a commencé avec une erreur impossible à comprendre : "aucune information de connexion valide trouvée." Pas de bouton retour, pas d'explication claire. Mon réflexe a été d'aller corriger ça directement dans mon éditeur de code (VS Code), en tapant des commandes dans la fenêtre de terminal intégrée.

Sauf qu'en réalité, je n'étais pas dans un vrai terminal. J'étais resté coincé dans une sorte de chat avec un modèle d'IA, qui ressemble énormément à un terminal mais qui n'en est pas un. Je donnais des instructions à voix haute, en gros, sans que personne n'agisse réellement derrière. C'est comme demander à un GPS de te recalculer un itinéraire alors que tu es en train de parler dans le vide : ça a l'air de fonctionner, mais la voiture n'a pas bougé d'un centimètre.

J'ai donc "configuré" un fichier qui n'a jamais réellement été modifié. Et ça a semé une confusion qui m'a suivi pendant tout le reste de la matinée.

Le mot de passe qui a fui

En cherchant à débloquer la situation, j'ai fait une erreur classique : j'ai copié-collé une clé d'accès (un genre de mot de passe technique) directement dans une conversation, alors que cette clé avait de l'argent rattaché derrière. Résultat : par sécurité, elle a été automatiquement désactivée, sans prévenir. J'ai continué à m'arracher les cheveux à essayer de la faire fonctionner alors qu'elle était déjà morte.

La leçon est simple : une clé d'accès, c'est comme la clé de ta maison. Tu ne la prends pas en photo et tu ne l'envoies pas dans un message juste pour montrer à quelqu'un à quoi elle ressemble.

L'erreur qui s'est incrustée

Voilà le vrai problème, celui qui a fait déraper toute la matinée : à un moment, en suivant des instructions pour réparer la situation, j'ai fini par enregistrer une configuration qui pointait vers un fournisseur (OpenRouter) directement dans les réglages profonds de Hermès. Le souci, c'est que cette information s'est incrustée tellement profondément que même en supprimant les dossiers de configuration et en réinstallant l'application depuis zéro, Hermès continuait à revenir vers OpenRouter, comme un GPS buggé qui te ramène toujours à la même adresse même quand tu changes de destination.

J'ai dû littéralement désinstaller et réinstaller Hermès plusieurs fois depuis le terminal pour m'en sortir, en pensant à chaque fois "cette fois c'est repartie de zéro" — alors que non, une trace persistait quelque part.

Ce qui a fini par débloquer la situation

Au final, ce n'est pas l'interface visuelle qui m'a sauvé (elle restait buggée), mais deux ou trois commandes tapées directement dans le terminal, une fois que j'étais sûr d'être au bon endroit :

rm -rf ~/.hermes

Pour repartir vraiment de zéro, sans rien garder de l'ancienne configuration.

hermes model

Pour forcer Hermès à me demander explicitement quel fournisseur et quel modèle utiliser, plutôt que de deviner tout seul (et de se planter).

Ces deux commandes m'ont permis de sortir de la boucle infinie et de connecter enfin Ollama Cloud correctement.

Ce que je retiens

  • Avant de taper une commande, vérifie où tu es réellement. Un prompt qui ressemble à un terminal n'en est pas forcément un — si les réponses ressemblent à des explications plutôt qu'à des résultats, tu es probablement au mauvais endroit.
  • Ne partage jamais une clé d'accès en clair, même par accident, même dans un outil de confiance. Une fois exposée, considère-la comme grillée.
  • Quand une interface graphique bloque en boucle, la solution est souvent une commande simple plutôt que de s'acharner à cliquer.
  • Une configuration mal enregistrée peut survivre à une suppression de dossier ou une réinstallation. Si le même problème revient après un "reset", c'est le signe qu'il faut chercher plus profond.

🚀 Le Guide Pratique

Si tu es ici juste pour que ça marche : voilà les étapes exactes, sans la galère.

1️⃣ Créer un compte Ollama Cloud

  1. Va sur ollama.ai (ou ollama.com)
  2. Crée un compte avec ton email
  3. Valide ton email
  4. Dans le dashboard, crée une nouvelle clé API (copie-la dans un endroit sûr, style un gestionnaire de mots de passe)

2️⃣ Configurer Ollama Cloud dans VS Code

Ouvre VS Code et suis ces étapes :

  1. Installe l'extension Cline (ou Claude Code)
  2. Va dans les paramètres de l'extension
  3. Configure le fournisseur comme Ollama Cloud
  4. Colle ta clé API (celle que tu as copiée à l'étape précédente)
  5. Choisis ton modèle préféré (ex: llama-2-7b, neural-chat-7b, etc.)

3️⃣ Configurer Hermès Agent (optionnel)

Si tu veux aussi utiliser Ollama Cloud avec Hermès Agent :

hermes config

Puis réponds à ces questions :

  • Provider → ollama_cloud
  • API Key → (colle ta clé)
  • Model → (choisis un modèle)

⚠️ Pièges courants (et comment les éviter)

  • Clé exposée ? Tes dépenses explosent dans quelques minutes. Regénère une nouvelle clé immédiatement dans le dashboard Ollama.
  • Configuration buggée qui refuse de partir ? Supprime ton dossier de config : 
    rm -rf ~/.hermes
    Puis relance avec : 
    hermes model
  • VS Code ne reconnaît pas Ollama Cloud ? Redémarre VS Code complètement (ferme et rouvre).
  • Besoin de vérifier ta clé fonctionne ? Test simple dans le terminal : 
    curl -X POST https://api.ollama.cloud/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "Salut"}]}'

💰 Coûts et limites

  • Abonnement Ollama Cloud : 20$/mois pour un quotidien raisonnable (10-20 requêtes/jour)
  • Modèles disponibles : llama, neural-chat, mistral, et autres (check le site pour la liste complète)
  • Latence : généralement 2-5 sec par réponse (dépend du modèle et de la charge du service)
  • Limite de tokens : varie selon le plan — vérifie sur le dashboard

✅ Vérifier que c'est opérationnel

Une fois configuré, teste ça dans VS Code ou Hermès :

"Dis bonjour en français et en anglais, puis compte jusqu'à 5"

Si tu reçois une réponse complète sans erreur, tu es bon. Ollama Cloud fonctionne. 🎉
